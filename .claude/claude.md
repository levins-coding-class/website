# Claude Development Guide for Levins Coding Class

## Projekt-Übersicht

Dies ist eine **Docusaurus-Website** für einen Kinder-Programmierkurs. Die Seite ist im **docs-only mode** konfiguriert (keine Landing Page, Docs sind unter `/`).

**Kursleiter:** Levin Keller

## Quick Start

```bash
# IMMER zuerst ausführen!
npm install

# Entwicklungsserver
npm start

# Produktions-Build
npm run build

# Produktions-Build lokal testen
npm run serve
```

## Tech Stack

- **Docusaurus 3.8.1** (Static Site Generator)
- **React 19.0**
- **TypeScript 5.6**
- **Node.js 22+** (erforderlich)

## Wichtige Kurs-Policies

### Windows-Policy (STRENG!)

- ❌ **Windows ist NICHT zugelassen** - auch nicht mit WSL
- Windows-Nutzer MÜSSEN die Cloud-Lösung verwenden
- Keine Ausnahmen!

### Hardware-Optionen

**Option A: Dedizierte Hardware**
- ✅ Linux (stark empfohlen) oder macOS
- ❌ Kein Windows
- Hardware NUR für das Kind, KEINE Daten anderer Familienmitglieder
- Kursleiter benötigt vollständigen Admin-Zugriff

**Option B: Cloud-Lösung (Hetzner)**
- Pflicht bei Windows-Computern
- Pflicht bei geteilten Familien-Rechnern
- Ca. 7 EUR/Monat
- Kursleiter greift NUR auf Cloud-Server zu

### Datenschutz-Policy

Bei dedizierter Hardware:
- Gerät muss **ausschließlich** für den Kurs genutzt werden
- **Keine** persönlichen Daten von Eltern
- **Keine** Daten von Geschwistern
- **Keine** Accounts/Fotos/Dokumente anderer Familienmitglieder
- Falls nicht möglich → Cloud-Lösung ist Pflicht

## Konfigurationsdetails

### Docs-Only Mode
- `routeBasePath: '/'` in docusaurus.config.ts
- Keine Landing Page
- Homepage ist `docs/intro.md` mit `slug: /`
- Blog ist deaktiviert

### Sprache
- Default Locale: `de` (Deutsch)
- Eltern-Bereich: informelles "Du"
- Alle Inhalte auf Deutsch

### URL-Struktur
- `/` → Homepage (docs/intro.md)
- `/eltern/...` → Eltern-Dokumentation
- `/schueler/...` → Schüler-Dokumentation
- `/lehrkraefte/...` → Lehrkräfte-Dokumentation

**IMMER** Docusaurus Admonitions für Hinweise:
```markdown

Text hier
:::

:::tip Tipp
Text hier
:::

:::warning Warnung
Text hier
:::

:::danger Wichtig
```

**NIEMALS** Bold-Text wie `**Hinweis:**` verwenden!

## Häufige Aufgaben

### Neue Dokumentation hinzufügen

1. Markdown-Datei im passenden Ordner erstellen
2. Frontmatter mit `sidebar_position` hinzufügen:
```yaml
---
sidebar_position: 4
---
```
3. Links mit absoluten Pfaden verwenden

### Kaputte Links finden

```bash
npm run build
```
Docusaurus schlägt bei kaputten Links fehl.

### Navigation ändern

- Navbar: `docusaurus.config.ts` → `themeConfig.navbar`
- Footer: `docusaurus.config.ts` → `themeConfig.footer`
- Sidebars: automatisch generiert aus Ordnerstruktur

## Pre-Push Checklist

**KRITISCH: Vor jedem `git push`:**

1. Remote-Änderungen holen:
   ```bash
   git fetch origin
   git pull origin main --no-rebase --no-edit
   ```

2. Dependencies installieren (falls package.json geändert):
   ```bash
   npm install
   ```

3. Build testen:
   ```bash
   npm run build
   ```
   - Build MUSS erfolgreich sein
   - Kaputte Links oder Fehler VOR dem Push fixen

4. Erst dann pushen:
   ```bash
   git push -u origin <branch-name>
   ```

**Niemals kaputte Builds pushen!**

## Terminology Guide

### Lernkurve (Learning Curve)

**Wichtig:** Eine **steile Lernkurve** bedeutet, dass etwas EINFACH zu lernen ist (schneller Fortschritt).
Eine **flache Lernkurve** bedeutet, dass etwas SCHWER zu lernen ist (langsamer Fortschritt).

Die meisten Menschen verwenden das falsch! Bei Beschreibungen von Schwierigkeit:
- "flache Lernkurve" = herausfordernd
- "steile Lernkurve" = schnell zu lernen
